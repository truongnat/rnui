## Design Context

### Users
React Native developers and UI designers who are building mobile applications. They use the docs during coding sessions to look up component APIs, during research phases to evaluate the library, and when demoing to teams to showcase capabilities.

### Brand Personality
Technical, modern, developer-focused. The interface should convey expertise and reliability while feeling approachable and well-organized.

### Aesthetic Direction
Reference large technical documentation sites like Vercel, Stripe, and Linear - clean, professional, with strong typography and thoughtful spacing. Support both dark and light modes with smooth transitions. The design should feel like a premium developer tool - precise, intentional, and not overly decorative.

### Design Principles
1. **Clarity over decoration**: Information hierarchy should be immediately obvious
2. **Developer-first**: Content and functionality take precedence over visual flourishes
3. **Professional polish**: Attention to spacing, typography, and micro-details without being flashy
4. **Accessible by default**: Good contrast, readable text, and keyboard navigation where applicable
