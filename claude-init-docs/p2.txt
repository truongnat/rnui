# Báo cáo hướng đi: React Native UI Design System Framework

## Tổng quan chiến lược

Mục tiêu là xây dựng một framework theo mô hình **dual-layer**: tầng dưới là headless (logic, accessibility, state hoàn toàn tách biệt style), tầng trên là styled default có thể override toàn bộ. Người dùng dùng headless để tự thiết kế từ đầu, hoặc dùng styled layer với theming token để có UI nhất quán ngay lập tức.

## Dual-layer chi tiết

Đây là trái tim của framework. Headless và Styled tồn tại song song — người dùng chọn mức nào cũng được, thậm chí mix cả hai.---

## 1. Theme Engine — trụ cột của sự đồng bộ

Toàn bộ giao diện đồng bộ dựa vào **token system** theo 3 tầng:

```
Primitive tokens  →  Semantic tokens  →  Component tokens
  gray.800           color.text.primary    button.bg.default
  blue.500           color.action.primary  input.border.focus
  spacing.4          radius.md             card.padding
```

Token được inject qua `ThemeProvider` dùng React Context + `useTheme()` hook. Styled components tiêu thụ semantic tokens, không bao giờ dùng primitive trực tiếp. Điều này đảm bảo khi switch theme (dark mode, brand override, accessibility scale) toàn bộ hệ thống cập nhật nhất quán.

```tsx
// ThemeProvider inject toàn bộ tokens
<ThemeProvider theme={myBrandTheme}>
  <App />
</ThemeProvider>

// Headless — không có token, chỉ logic
const { isPressed, handlers } = usePressable({ onPress });

// Styled — dùng tokens qua useTheme
const Button = ({ variant }) => {
  const { tokens } = useTheme();
  return <Pressable style={tokens.button[variant]} {...handlers} />;
};
```

## 2. Hiệu năng — các quyết định kỹ thuật quan trọng

Đây là điểm khác biệt lớn nhất so với các thư viện hiện có:

**Không StyleSheet.create toàn cục.** Thay bằng style object được memo hóa theo theme — chỉ re-compute khi token thay đổi, không phải mỗi render.

**Reanimated 3 cho tất cả animation.** Không dùng `Animated` API cũ. Mọi transition chạy trên UI thread, không block JS thread. Định nghĩa `entering`/`exiting` một lần ở token level:

```tsx
// Animation token
const motionTokens = {
  press: withSpring({ damping: 18, stiffness: 300 }),
  enter: FadeInDown.springify(),
  exit: FadeOutUp.springify(),
};
```

**Worklet-first gesture handling** qua Gesture Handler v2. `onPress`, `onSwipe`, `onPinch` đều chạy trên UI thread.

**FlashList thay FlatList** cho tất cả list components. FlashList tái sử dụng cell theo type, giảm mount/unmount đáng kể.

**Lazy loading styled components.** Mỗi component là module riêng, tree-shake được. App chỉ bundle những gì dùng thật sự.

## 4. Stack kỹ thuật được đề xuất

**Core runtime:**
- `react-native` 0.83+ (New Architecture bắt buộc)
- `react-native-reanimated` 3.x — tất cả animation
- `react-native-gesture-handler` 2.x — gesture trên UI thread
- `@shopify/flash-list` — list hiệu năng cao

**Build & tooling:**
- Monorepo với `pnpm workspaces` + `turborepo` — build cache, parallel tasks
- `expo-modules-core` cho native module nếu cần
- `tsup` để bundle — output ESM + CJS + type declarations
- `changeset` để quản lý versioning

**Testing:**
- `jest` + `@testing-library/react-native` cho unit/component
- `Detox` cho E2E trên simulator thật
- `reassure` cho performance regression testing — đây là điểm then chốt để đảm bảo không có render regression khi release

**Docs:**
- `Storybook for React Native` v8 — chạy trực tiếp trên device
- `Docusaurus` cho website docs + API reference tự động từ TSDoc

## 5. Những rủi ro cần chú ý sớm

**Token sprawl:** Token system dễ bị phình ra nếu không có convention chặt. Đề xuất giới hạn cứng: tối đa 3 tầng (primitive → semantic → component), không được dùng primitive token trực tiếp trong component — enforce bằng ESLint custom rule.

**Styled vs Headless drift:** Khi codebase lớn, styled component dễ bị copy logic thay vì wrap headless. Cần thiết lập test rule: mọi styled component phải import ít nhất 1 headless hook từ package headless.

**Reanimated complexity:** Shared values và worklet scope dễ gây memory leak nếu không cleanup đúng. Tạo `useAnimatedStyle` wrapper chuẩn cho toàn framework, không để từng component tự manage.

**Android/iOS divergence:** Một số gesture (swipe-to-dismiss, pull-to-refresh) behave khác nhau native. Thiết lập cross-platform test suite từ Phase 1, không để đến Phase 3 mới phát hiện.
