# 🧑‍💼 Day 6/100 — Technical Documentation (nâng cao)
**Date:** 2026-03-21

## Concept cốt lõi
1. **Docs là code thứ hai** — Nếu chỉ mình bạn hiểu → team tê liệt khi bạn vắng
2. **Ghi WHY, không chỉ WHAT** — ADR là công cụ audit quyết định
3. **Level hóa audience** — Developer / Tech Lead / Manager
4. **Living docs** — Gắn docs review vào PR/sprint
5. **Diagram > Words** — C4 Model là chuẩn vàng

## C4 Model
Context → Container → Component → Code

## README Template cho service
```markdown
# [Service Name]
## 📌 Purpose / 🚀 Quick Start / ⚙️ Configuration / 📝 ADRs
```

## Insight
> "The most senior engineers spend 30–40% of their time writing — docs, proposals, ADRs. Code is temporary, but decisions documented outlast both the code and the coder."

**1 ADR/tuần × 1 năm = team mới onboard trong 2 ngày thay vì 2 tuần.**
