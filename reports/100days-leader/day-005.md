# 🧑‍💼 Day 5/100 — Viết Technical Documentation
**Date:** 2026-03-20

## Concept cốt lõi
1. **Docs là sản phẩm** — Doc tốt = team không cần hỏi bạn mỗi ngày
2. **Audience-first** — Developer / Tech Lead / Manager cần detail khác nhau
3. **Pyramid structure** — TL;DR trước, chi tiết sau
4. **Living docs > Perfect docs** — Doc outdated nguy hiểm hơn không có doc
5. **Ghi quyết định** — WHY quan trọng hơn WHAT

## ADR Template
```
# ADR-001: [Tên quyết định]
## Status: Accepted
## Context / Decision / Consequences / Alternatives Considered
```
