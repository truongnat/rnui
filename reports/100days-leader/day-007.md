# 🧑‍💼 Day 7/100 — Kỹ Thuật Viết Technical Doc Hiệu Quả
**Date:** 2026-03-22

## 🧠 Concept cốt lõi

1. **Doc tốt = Code tốt** — Architect được đánh giá qua khả năng viết rõ, không kém gì qua code
2. **Pyramid (SCQA)** — Situation → Complication → Question → Answer. Bắt đầu từ kết luận, chi tiết theo sau
3. **3 loại doc quan trọng nhất** — RFC, ADR, Runbook — mỗi loại có mục đích và đối tượng khác nhau
4. **Audience-first** — Ai đọc? Họ cần quyết định gì từ doc này?
5. **Living doc** — Ghi rõ ngày viết, phiên bản, ai maintain

## ✍️ ADR Template

```
# ADR-001: [Tên quyết định]
Ngày: YYYY-MM-DD | Trạng thái: Accepted

## Bối cảnh / Quyết định / Hệ quả / Alternatives đã cân nhắc
```

## 💬 Templates

**RFC email:**
```
Subject: [RFC] Đề xuất X

TL;DR: [1-2 câu]
Cần feedback trước [ngày X]:
- [ ] Blocker kỹ thuật nào bỏ sót?
- [ ] Trade-offs quan trọng chưa đề cập?
```

**PR description:**
```
## What / Why / How / Testing
```

Senior → Staff/Principal không phải vì code giỏi hơn, mà vì **communicate decisions** tốt hơn.
