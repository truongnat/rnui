# 🎯 Day 7/100 — Hexagonal Architecture (Ports & Adapters)
**Date:** 2026-03-22

## 📚 Lý thuyết cốt lõi

- **Hexagonal Architecture** (Alistair Cockburn, 2005) — tách hoàn toàn business logic khỏi mọi infrastructure concern (DB, HTTP, queue, UI)
- **Domain core** ở trung tâm: entities, use cases, pure business logic — không import bất kỳ framework nào
- **Port** = interface do domain định nghĩa: "Tôi cần ai đó làm điều này" (`UserRepository`, `EmailService`, `PaymentGateway`)
- **Adapter** = implementation cụ thể của Port (`PostgresUserRepository`, `SendGridEmailAdapter`) — sống ở ngoài, domain không biết chúng tồn tại
- **Driving side (Left)** = adapters kích hoạt app (HTTP controller, CLI, test)
- **Driven side (Right)** = adapters được app kích hoạt (DB, email, external APIs)
- **Dependency rule**: luôn từ ngoài → trong. Adapter phụ thuộc Port. Domain không phụ thuộc ai.

## 💡 TypeScript Example

```typescript
// DOMAIN — zero external dependencies
export class User {
  verify(): void {
    if (this._isVerified) throw new Error('User already verified');
    this._isVerified = true;
  }
}

// PORTS — defined by domain
export interface UserRepository {
  findById(id: string): Promise<User | null>;
  save(user: User): Promise<void>;
}

// USE CASE — chỉ phụ thuộc ports
export class VerifyUserUseCase {
  constructor(private userRepo: UserRepository, private emailService: EmailService) {}
  async execute(userId: string, token: string): Promise<void> {
    const user = await this.userRepo.findById(userId);
    if (!user) throw new Error('User not found');
    user.verify();
    await this.userRepo.save(user);
    await this.emailService.sendVerificationEmail(user.email, token);
  }
}

// IN-MEMORY ADAPTER — cho test nhanh, không cần DB
export class InMemoryUserRepository implements UserRepository {
  private store = new Map<string, User>();
  async findById(id: string) { return this.store.get(id) ?? null; }
  async save(user: User) { this.store.set(user.id, user); }
}

// TEST — zero infrastructure!
it('should verify user', async () => {
  const repo = new InMemoryUserRepository();
  const emailSvc = { sendVerificationEmail: jest.fn() };
  const useCase = new VerifyUserUseCase(repo, emailSvc);
  repo.seed(new User('u1', 'user@test.com', 'Truong', false));
  await useCase.execute('u1', 'token-123');
  expect((await repo.findById('u1'))!.isVerified).toBe(true);
});
```

## 💎 Insight

> **"Hexagonal Architecture không phải là về thư mục — nó là về dependency direction."**

**Test là litmus test tốt nhất**: nếu không thể test business logic mà không khởi động infrastructure → domain chưa thực sự isolated. Mỗi lần cần mock `prisma.user.findUnique` trong unit test = domain đang bị leak.

Architect nhìn Hexagonal như **organizational boundary** — ports là contract giữa team platform (infra) và team domain (business), cho phép hai team làm việc độc lập.
